# CS 230 Project
The CS 230 project was a project to design software for a client called "The Gaming Room". The Gaming Room wanted me to design software for their game called "Draw It or Lost It". They wanted software that can host their draw it or lose it game across multiple operating platforms. 
I think I did a good job evaluating requirements and design elements for the software like the requirements, design constraints, and the domain model. 
When I developed the code I found the constraints, and the UML diagram to be the most useful, those allowed me to make sure the code reflected on exactly how the project was intended to look. If I could revise one part of the code it would be the system architecture view. I didn't see that it wasn't required until now and I completely didn't think to go back and add something for it. I implemented the users needs into the design document by adding the proper requirements and design constraints into the document. This allowed me to return back and make sure those parts were implemented correctly. My approach for designing this software was to consider all the aspects that were required before I made final decisions. Then when I had all the information, I would make decisions based on all the factors and what designs would accomodate the conflicting parts of the software.
